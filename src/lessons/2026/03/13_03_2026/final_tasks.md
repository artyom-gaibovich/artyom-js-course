# 5. Объектно-ориентированное программирование

---

### Задача1. Объект пользователя

**Условие**

Создайте объект `user`, который хранит информацию о пользователе.

Объект должен содержать:

**Свойства**

```
name
age
email
```

**Методы**

`getInfo()`
возвращает строку с информацией о пользователе.

`changeEmail(newEmail)`
изменяет email пользователя.

---

### Входные данные

```javascript
const user = {
 name: "Alex",
 age: 25,
 email: "alex@gmail.com"
}
```

Вызовы:

```
user.getInfo()
user.changeEmail("alex123@gmail.com")
user.getInfo()
```

---

### Ожидаемый результат

```
Alex (25) - alex@gmail.com
Alex (25) - alex123@gmail.com
```

---

### Задача2. Банковский счет

**Условие**

Создайте объект `bankAccount`.

Объект должен хранить баланс пользователя и позволять выполнять операции с деньгами.

---

### Свойства

```
owner — имя владельца
balance — текущий баланс
```

---

### Методы

`deposit(amount)`
пополняет баланс на указанную сумму.

`withdraw(amount)`
снимает деньги со счета.

Если денег недостаточно — вывести сообщение:

```
Недостаточно средств
```

`getBalance()`
возвращает текущий баланс.

---

### Входные данные

```javascript
account.deposit(500)
account.withdraw(200)
account.getBalance()
```

---

### Ожидаемый результат

```
Баланс: 300
```

---

### Задача3. Корзина интернет-магазина

---

### Свойства

```

```

# 8. SOLID

---

### Задача1. Single Responsibility

**Условие**

Есть класс:

```javascript
class UserManager {
 createUser(){}
 saveUser(){}
 sendEmail(){}
}
```

Этот класс нарушает принцип **Single Responsibility**.

---

### Нужно:

Разделить его на два класса.

---

### Класс 1

```
UserService
```

Методы:

```
createUser()
saveUser()
```

---

### Класс 2

```
EmailService
```

Метод:

```
sendEmail()
```

---

### Ожидаемый результат

Код должен быть разделен на два независимых класса.

---

### Задача2. Dependency Injection

**Условие**

Есть класс `OrderService`.

Он отправляет email после создания заказа.

Плохая реализация:

```javascript
class OrderService {
 constructor(){
   this.emailService = new EmailService()
 }
}
```

---

### Нужно

Передать `EmailService` **через конструктор**.

---

### Входные данные

```javascript
const emailService = new EmailService()
const orderService = new OrderService(emailService)
```

---

### Ожидаемый результат

Класс `OrderService` должен использовать переданную зависимость.

---
